Contexte

Problematique

Les normes 3GPP (4G, 5G, 6G) generent une documentation technique enorme et complexe. Les ingenieurs peinent a trouver rapidement l'information pertinente.

Solution proposee

Un systeme de Question-Reponse intelligent combinant recherche documentaire et intelligence artificielle pour repondre instantanement aux questions techniques.

Dataset

TeleQnA: 1461 questions + 366 de test sur les normes 3GPP (Releases 15-18)

Objectif

Comparer 6 approches de Q&A sur des criteres objectifs (precision, cout, temps)

Impact

Accelerer la formation et le travail des ingenieurs telecom

Analyse comparative

Forces

  • FT + RAG: Precision maximale (F1=0.86)
  • RAG simple: Excellent rapport simplicite/performance
  • Hybrid: Meilleur recall (0.80)
  • Fine-tuning: Pas de cout API recurrent

Faiblesses

  • LLM seul: Trop d'hallucinations (F1=0.37)
  • FT + RAG: Maintenance complexe
  • Reranking: Temps de traitement +1.3s
  • Fine-tuning: Cout GPU eleve

Recommandations

Cas d'usageApproche recommandeeJustification
POC / Projet academiqueRAG simpleImplementation rapide, resultats solides
Production (cout maitrise)RAG + Hybrid + RerankRobuste, maintenable, sources citables
Expertise domaine maximaleFine-tuning + RAGPrecision maximale pour applications critiques
Budget API limiteFine-tuning seulPas de cout recurrent, inference locale

A retenir

Le RAG simple est l'approche recommande pour demarrer. Il offre un excellent equilibre entre performance (F1=0.67) et simplicite de mise en oeuvre. La combinaison FT+RAG est recommandee pour les systemes critiques necessitant une precision maximale.

Conclusion

Resultats cles
